# ISPW2 Master's Degree Project – Falessi Module

## Overview

This repository contains the project developed for the **ISPW2 (Software Engineering and Project Management)** course of the Master's Degree program, specifically for the **Falessi module**.

The project focuses on an empirical study of the open-source project **Apache Storm**. The goal is to investigate how software quality metrics and machine learning techniques can support software development activities.

---

## Project Objectives

The main objectives of this project are:

1. Analyze the Apache Storm open-source project.
2. Extract and collect a dataset containing selected software metrics.
3. Assign **Buggy / Non-Buggy** labels to software classes.
4. Train and evaluate prediction models on the generated dataset.
5. Assess the potential contribution of predictive models to the software development process.

---

## Dataset Creation

A dedicated dataset is created from the Apache Storm project by extracting specific software engineering metrics from the source code.

The dataset includes:

- Software quality metrics
- Class-level information
- Bug-related data
- Buggy / Non-Buggy labeling

This dataset serves as the foundation for the prediction and evaluation activities performed in the study.

---

## Defect Prediction Study

Using the generated dataset, machine learning models are trained to predict whether a class is likely to be:

- **Buggy**
- **Non-Buggy**

The study evaluates the effectiveness of these models and analyzes their predictive capabilities.

---

## Contribution to Software Development

The final goal of the project is to verify the potential value that defect prediction models can provide within a software development lifecycle.

In particular, the study investigates how predictive models may:

- Support developers in identifying high-risk classes.
- Improve testing and quality assurance activities.
- Help prioritize maintenance efforts.
- Contribute to more informed development decisions.

---

## Case Study

**Target Project:** Apache Storm

Apache Storm is used as the case study for collecting data, generating the dataset, and evaluating defect prediction models.
